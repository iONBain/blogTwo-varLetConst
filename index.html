blogBhaskar | Infamous discussion on var, let, const in JS

Main purpose of this blog is to clear my understanding of the difference between var, let and const. These data-types are sometimes confusing and Let's finally put an end to this for me as well as for yourself.

Headings discussed:- scope, hoisting, re-initialization and re-assigning (updation)

Scope: Scope in simple terms means where these variables are accessible. The limit of the availability of a particular variable. Block scoped variables are better than the globally scoped as they don't update/refer to other same name variables.

Hoisting: It means that we can define a variable before its declaration. Accessing variables in which hoisting is allowed gives a value undefined. If a declaration type doesn't allow hoisting, then Reference-Error will be thrown.

Re-initialization: Re-initialization of a variable means that you can initialize the variable again. var is the only declaration which allows this feature, because it was there before ES6.

Re-assigning: Re-assigning is basically updation of variables, to reassign the value of a variable. Just the const declaration type doesn't allow the re-assigning feature.

Headings discussed:- var, let, const

var:

In short, global/function scoped, can be redeclared and updated, and are hoisted to the top of their scope with undefined as the initial value.

It is also the most used declaration type of variables. Before ES6, var was used. However, some issues are associated with var variables, and we will be looking into those, but let's first discuss other declaration variable types on a high level.

let:

It is a block scoped, can be updated but cannot be re-initialised. It is also hoisted to the top of their scope, but no initial value is defined.

It is the improvement to var declaration and also solves the issues of the var type declaration.

const:

Everything similar to let declaration, but cannot be updated. So, block scoped, cannot be updated neither re-intialised. Hoisted to the top of their scope, but with no initial value.

Mainly used for constants, like fixed URL to fetch, or some fixed numbers such as "pie","g" or "e".

Summarized var, let, const in tabular format as follows:-

Parameters var let const
Scope Global/ Function Block Block
Hoisting At the top of their scope Not Allowed Not Allowed
Re-assigning Yes Yes No
Re-initialization Yes No No
